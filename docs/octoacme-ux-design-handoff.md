## Purpose
Ensure smooth collaboration between UX Designers and Developers, with clear design specifications and acceptance criteria for user-facing features.

## When to Use
- Any feature with new or significantly changed user interface elements
- User workflows that require usability consideration
- Updates to existing UI that affect user experience

## UX Design Workflow

### 1. Discovery & Research
- Collaborate with Product Manager on user needs and requirements
- Conduct user research if needed (interviews, surveys, analytics review)
- Review Customer Support Liaison feedback on user pain points
- Document design goals and success criteria

### 2. Design & Iteration
- Create wireframes for early feedback
- Develop high-fidelity mockups and prototypes
- Conduct design reviews with stakeholders
- Iterate based on feedback from Product Manager and team

### 3. Design Handoff
- Prepare design specifications and assets
- Document interaction patterns and edge cases
- Share design files in accessible location (e.g., Figma, Sketch)
- Schedule handoff meeting with developers

### 4. Development Support
- Attend sprint planning to clarify design requirements
- Review PRs for design fidelity
- Answer questions during implementation
- Validate design in staging environment

### 5. Validation & Refinement
- Conduct usability testing on staging or beta releases
- Gather feedback from Customer Support Liaison
- Document design decisions and learnings
- Iterate on design improvements for future releases

## UX Design Handoff Checklist

- [ ] User research findings documented (if applicable)
- [ ] Wireframes and high-fidelity mockups completed
- [ ] Design specifications include:
  - [ ] Layout and spacing measurements
  - [ ] Typography and color specifications
  - [ ] Interactive states (hover, active, disabled, error, etc.)
  - [ ] Responsive behavior for different screen sizes
  - [ ] Accessibility requirements (ARIA labels, keyboard navigation, contrast ratios)
- [ ] Design assets exported and shared with developers
- [ ] Edge cases and error states documented
- [ ] User flow diagrams provided (if applicable)
- [ ] Handoff meeting scheduled with development team
- [ ] Design review scheduled for implementation validation
- [ ] Acceptance criteria updated in backlog items with design requirements

## Design Review Criteria

When reviewing implementation:
- Visual accuracy matches design specifications
- Interactive states function as designed
- Responsive behavior works across target devices
- Accessibility requirements are met
- User flow is intuitive and matches design intent
- Edge cases are handled gracefully

## Collaboration Best Practices

- Keep design files and specs in a central, accessible location
- Document design decisions and rationale
- Be available during development for questions
- Provide timely feedback on PRs with UI changes
- Participate in demos and sprint reviews
- Share user feedback and learnings with the team
